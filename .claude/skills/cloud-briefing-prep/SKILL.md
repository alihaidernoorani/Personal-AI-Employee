# Cloud Briefing Prep

Aggregate operational data from the cloud VM into a structured briefing data file
for consumption by the ceo-briefing skill on the local machine.

---

## Step 1 — Validate Cloud Boundary

1. Verify `AGENT_ROLE=cloud` is set. If not: output "ERROR: cloud-briefing-prep only runs on AGENT_ROLE=cloud" and stop.
2. Import `safe_vault_write` from `watchers/cloud_boundary.py`.
3. All vault writes MUST use `safe_vault_write()`.

---

## Step 2 — Read Signal Data (Last 7 Days)

Scan `AI_Employee_Vault/Signals/` for all `SIGNAL_*.md` files.

For each file:
- Read YAML frontmatter: `signal_type`, `severity`, `created`, `requires_human_action`
- Include only files where `created` is within the last 7 days

Compute:
- `total_signals`: count of all signals in window
- `signals_by_type`: dict of signal_type → count
- `signals_by_severity`: dict of severity → count (`info`, `warning`, `critical`)
- `requires_human_action_count`: count where `requires_human_action: true`
- `signal_summary`: top 3 signal types by frequency

---

## Step 3 — Read Uptime from Heartbeat Files

List all `Updates/HEARTBEAT_*.md` files from the last 7 days.

For each file, read: `created`, `watcher_status`, `tasks_in_progress`

Compute:
- `heartbeat_count_7d`: total heartbeats in window
- `expected_heartbeats`: (7 days × 24 hours × 60 min) / 5 = 2016 expected at 5-min intervals
- `cloud_agent_uptime_pct`: (heartbeat_count_7d / expected_heartbeats × 100), capped at 100%
- `watcher_status_summary`: most recent `watcher_status` dict from newest HEARTBEAT file

If no HEARTBEAT files exist: set `cloud_agent_uptime_pct: 0`, note "No heartbeat data"

---

## Step 4 — Read Vault Health Data

Find the most recent `Updates/VAULT_HEALTH_*.md` file (sort by filename timestamp).

Read:
- `health_status` from YAML frontmatter
- `sync_lag_seconds`
- `stale_tasks_count`
- File creation timestamp

If no VAULT_HEALTH file exists within last 7 days: set `vault_health_status: "unknown"`.

---

## Step 5 — Read Social Post Audit Entries

Scan `AI_Employee_Vault/Logs/` for today's and last 7 days' `.json` files.

For each log entry where `action_type` contains "social_post":
- Extract: `timestamp`, `target` (platform), `result`
- Count by platform: linkedin, facebook, instagram, twitter
- Count by result: success, failure

Compute:
- `social_posts_7d`: total posts in window
- `social_posts_by_platform`: dict of platform → count
- `social_post_success_rate`: (success_count / total × 100)%

---

## Step 6 — Write Briefing Data File

Write `AI_Employee_Vault/Updates/BRIEFING_DATA_<YYYY-MM-DD>.md` using `safe_vault_write()`:

```markdown
---
type: briefing_data
generated_by: cloud_agent
generated_at: <ISO-8601Z>
date: <YYYY-MM-DD>
cloud_agent_uptime_pct: <float>
vault_health_status: <healthy|degraded|critical|unknown>
sync_lag_seconds: <int>
stale_tasks_count: <int>
total_signals_7d: <int>
requires_human_action_count: <int>
social_posts_7d: <int>
social_post_success_rate: <float>
---

# Cloud Briefing Data — <YYYY-MM-DD>

*Generated by cloud agent at <timestamp>. Consumed by ceo-briefing skill on local machine.*

## Cloud Agent Uptime (Last 7 Days)

- **Uptime**: <uptime_pct>% (<heartbeat_count_7d> / <expected_heartbeats> heartbeats)
- **Watcher Status**: <watcher_status_summary>

## Vault Health

- **Health Status**: <vault_health_status>
- **Sync Lag**: <sync_lag_seconds>s
- **Stale Tasks**: <stale_tasks_count>

## Signal Summary (Last 7 Days)

| Signal Type | Count |
|-------------|-------|
| <type> | <count> |
...

**By Severity**: Critical: <n>, Warning: <n>, Info: <n>
**Requires Human Action**: <requires_human_action_count>

## Social Post Activity (Last 7 Days)

| Platform | Posts |
|----------|-------|
| LinkedIn | <n> |
| Facebook | <n> |
| Instagram | <n> |
| Twitter | <n> |

**Success Rate**: <success_rate>%
```
